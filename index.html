微信小程序使用webview

##前提条件
1、备案域名

2、http修改为https

3、微信公众平台后台配置小程序业务域名#企业级开发者才能配置小程序业务域名

##域名备案
涉及的服务商有三个：阿里云、西部数码

涉及的业务是，客户使用的是在西部数码购买的域名，而项目使用的是阿里云的服务器，在西部数码的域名解析界面配置a记录，添加二级域指向阿里云地址时，出现dns服务器缓存的页面。

图片

域名已备案，但是工信部要求，域名服务商要和主机商相同，即需要讲备案接入阿里云，二级域才能解析到阿里云的服务器。

阿里云接入时的提示是：接入只是添加一个接入信息，不影响原来网站的使用。

只要接入通过初审后，dns解析就已经生效了，就可以使用了，但后续步骤还是要完成的。

阿里云接入备案流程

##http改为https
将http改为https，需要的就是ssl证书，我使用过的是华为云和阿里云的ssl，其实配置都差不多。

1、申请ssl证书

阿里云可以购买免费证书，在购买ssl证书页面中选择Symantec品牌，其中有免费型DV SSL类型，按要求流程操作就行。

在域名管理界面，添加对应的域名的text类型的DNS信息。

2、配置ssl证书

在阿里云管理控制台中，进入ssl证书管理界面，下载对应的ssl证书，并将ssl证书上传到服务。

在centOS-apache下配置ssl证书如下：（阿里云、华为云的配置方法有点老了）

1、安装opnessl

2、不需要修改/etc/httpd/conf/httpd.conf

解释：

查看文件发现最后一行有IncludeOptional conf.d/*.conf，这已经将Include conf/extra/httpd-ssl.conf包含了

同时 Include conf.modules.d/*.conf也将LoadModule ssl_module modules/mod_ssl.so包含了，这两项无须配置

3、修改/etc/httpd/conf.d/ssl.conf

按阿里云文档配置就行了

配置完成后重启apache

4、最坑的点来了，当你以为这样去访问时https站点时，其实会发现无法连接到服务器。

检查一下检查以下问题：

在检查以上都没问题后，最后使用站长工具对阿里云服务器端口扫描发现，发现443端口是关闭的，百度后发现，要在阿里云服务器的控制台添加安全组规则，将443端口加上，就可以使用https访问了

##配置小程序业务域名

登录小程序后台，设置->开发设置->添加业务域名,下载文件，将文件放在服务器/var/www/html目录下。

Mac安装hexo-github-blog

安装环境HomeBrew、npm、node

1.安装HomeBrew

  • curl: (7) Failed to connect to raw.githubusercontent.com port 443: Operation timed out//连不上外网
  • 使用ss后报错
    curl: (35) Server aborted the SSL handshake
    参考博客

2.安装node、更新npm

3.安装git

4.安装hexo

配置

1.生成ssh_key

2.在github setting中配置对应的ssh_key_pub

  • github右上角用户图标 => setting => SSH and GPG keys => New SSH Key

3.测试ssh配置是否成功

4.创建github repo

  • repo name： (ower name).github.io

5.配置hexo项目

tips

  • 在_config.yml中名/值中要有空格例如：type：（空格）git
  • 在第一次hexo d的之前，要有npm install hexo-deployer-git –save

  • hexo new page “pageName” #新建页面,还不清楚new post与new page有什么区别
